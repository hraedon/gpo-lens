## On the project

The codebase continues to be well-structured. The architecture boundary (core never imports web/narration) held up through all changes — the Protocol in model.py flows cleanly to danger.py, queries.py, detection.py, and web/app.py without crossing the boundary. The import direction (web → model) is the allowed direction.

The `_localname` consolidation (normalize.py) and `SEVERITY_ORDER` centralization (model.py) were pre-existing uncommitted work from a prior session. These are good DRY fixes but the diff was noisier than expected because of them.

## On the work done

The AdmxResolver Protocol is the cleanest change — it replaces 8 duck-typed `object | None` call sites with a proper Protocol contract. The dual adversarial review caught real issues: incomplete migration (detection.py and queries.py:baseline_diff still used PolicyDefinitions), dead-code `getattr` patterns that undermined the Protocol, and a consolidation gap (`_SEVERITY_RANK` in web/app.py). All were fixed.

The danger calibration tests (WI-032) revealed an important finding: the 5 TOML Bucket 1 rules produce zero findings against both sample estates because the estates use ADMX-managed registry settings (`Registry:Policy:*` identities), not raw `HKLM\...` registry preferences. This is expected behavior, not a rule defect — but it means the TOML rules have never been validated against an estate that actually has registry preferences. The calibration tests document this honestly.

The `_get_rw_conn` helper is a pragmatic fix for the DB connection lifecycle issue. Two reviewers flagged that it calls `_store._restrict_db_permissions` (a private API). Acceptable for now, but the cleaner path is to move the helper into `store.py` as a public function.
