## On the project

The codebase is in excellent shape. Architecture is clean — import boundaries enforced by AST-based test, deterministic core has zero AI in the truth path, and the module separation is disciplined. The test suite is substantial (1310 tests, ~18,800 lines) and covers both synthetic fixtures and real-estate calibration. The `danger.py` + `danger_rules.toml` system is well-designed (cited, TOML-driven Bucket 1 rules + pure-function Bucket 2 detectors), though it lacks the calibration tests that B.5 demands. The web UI is more surface area than a CLI-only tool needs, but it's well-tested and the CSRF/auth hardening is thorough.

What feels fragile: `queries.py` at 1093 lines is a re-export hub that depends on nearly everything. It's the "god module" — not broken, but a coupling risk. The `store.py` schema migration system (additive, PRAGMA user_version) works but has no rollback path. The `merge.py` module at 982 lines is doing a lot — CSE merge modes, token building, security-gate eval, principal resultant — and the naming overlap between `merge.py` (principal resultant) and `topology.py` (settings-at-som) could confuse newcomers.
