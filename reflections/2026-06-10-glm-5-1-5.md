# Session Reflection — 2026-06-10

**Work summary:** Implemented three pending features from active breadcrumbs: settings-diff pipeline, delegation-audit-deep-dive (SDDL parsing, deny ACE detection, excessive-writer rollup), and estate-doc-export (markdown/HTML reports). Ran adversarial review on all three, then fixed 13 critical and 21 important issues found by reviewers. All breadcrumbs moved to resolved/.

---

## On the project

gpo-lens is shaping up well. The normalized data model in `model.py` is clean and serves as a solid contract. The spec-driven approach (docs/spec/wi_*.md) is paying dividends — each feature has clear acceptance criteria. The biggest design debt is in `cli.py`, which has grown into a 1400+ line file with one function per subcommand. It works, but it's getting unwieldy. The display module is also stretched — it started as a table renderer but now does grouped text rendering, diff display, etc.

The SDDL parsing was the trickiest addition. SDDL is a genuinely hostile format — section headers (O:, G:, D:, S:) can appear inside SID strings, rights are concatenated 2-letter codes without delimiters, and ACE types include object-type variants (OA, OD) that are common in AD GPOs. The adversarial reviewer caught all of these, which was satisfying.

## On the work done

The three-parallel-agent approach worked well for implementation. Each agent had full context from the breadcrumb specs and AGENTS.md conventions, and produced working code. The adversarial review round was high-value — it caught real bugs:

- **settings-diff**: Join key missing `side` column (silent data loss), unhandled ValueError/KeyError on malformed input, dead code in display.py
- **SDDL parsing**: Section-start detection corrupted owner SIDs when DACL was empty (the `D:` inside `S-1-5-18D:` was misread as a DACL header), missing Object ACE types (OA/OD), read-only rights LC/LO incorrectly in the write-rights set, concatenated rights like `RPWP` not parsed
- **report.py**: Three XSS vectors (unescaped domain in HTML table, unescaped severity in badges), missing Per-OU effective settings section per spec, crash on empty snapshot DB, silently dropped baseline "extra" findings

The fix agents handled all of these cleanly. The final state is ruff-clean, mypy-clean, and all tests passing.

What I'm most confident in: the SDDL parser (after the depth-tracking rewrite), the settings-diff join logic (now 4-tuple key with error handling), the report HTML escaping.

What I'd want a second pair of eyes on: the `_parse_sddl_rights` concatenated-rights parser in detection.py — it works for known 2-letter codes but the fallback (skip unknown character, advance 1) could misalign on genuinely malformed input. The report's Per-OU section calls `settings_at_som` for every SOM — this could be slow on large estates (hundreds of SOMs), but we optimized the O(n^2) away by passing the already-found SOM object.

## Gaps to flag

- `cli.py` is 1400+ lines and growing. Each new subcommand adds ~50 lines. Consider splitting into per-feature CLI modules.
- `report.py` generates markdown by string concatenation — no template engine (correct per stdlib-only constraint), but it's fragile. The `_SUMMARY_FIELDS` extraction helps, but the Per-OU and Per-GPO sections are still hand-built HTML.
- `detection.py:_parse_sddl_rights` falls back to single-character advancement on unknown codes. This is safe for known SDDL but could misparse truly malformed input. Consider logging a warning.
- `settings_diff` skips rows with invalid GUIDs silently. In a 10,000-row export, a single bad row is dropped with no indication. Consider a summary count of skipped rows.
- The `cmd_report` return type was changed from `None` to `int`, but other cmd_ functions still return `None`. Inconsistent — either all should return int or none should.
- `ExcessiveWriter` and `SddlAcl` now use `tuple` fields (immutable), but other model dataclasses still use `list`. Inconsistent mutability contract across model.py.
