## On the project

gpo-lens is the most architecturally disciplined project in the portfolio. The "no AI in the truth path" charter is not aspirational — it's enforced by architecture tests, and the scope honesty work (S.1-S.5) made the deterministic core *more* honest without breaking that constraint. The "flag, don't simulate" principle now covers loopback, security filtering, WMI filters, and item-level targeting. That's the right answer to the question every RSoP-adjacent tool gets wrong.

The complement-plan exercise clarified the portfolio strategy: gpo-lens is the deep, point-in-time, air-gapped analysis tool. The complements cover adjacent spaces (real-time monitoring, write-side remediation, compliance evidence, cross-estate comparison) without bloating gpo-lens. The strongest pairing is gpo-lens + a 5136 event daemon — together they answer "what is the estate" and "what just changed."

The decomposition of queries.py into topology.py (546 lines) was overdue but clean. The DAG topology.py → detection.py → model.py has no cycles, and backward-compatible re-exports mean zero downstream changes. queries.py at 1,394 lines is still large but defensible — it's the composition layer.

## On the work done

**What I'm confident in:**
- The scope honesty functions are charter-compliant. The adversarial reviewer caught a charter violation ("applied only to" = simulation claim) and I fixed it to "exclusivity not evaluated; default ACEs and group membership not modeled." That's honest.
- The security hardening (streaming decompression + CSRF tests) is real defense, not theater. The spoofed-header zip bomb is caught during decompression, not after.
- The narration dispatch-driven prompt is genuinely single-source-of-truth now. Adding a query to `_QUERY_DISPATCH` + `_QUERY_DESCRIPTIONS` is the only edit for `ask` routing.

**What I'd want a second pair of eyes on:**
- `is_security_filtered` is the function most likely to produce false positives in real estates. It checks only Authenticated Users and Domain Computers; `Everyone` (S-1-1-0) is not in the broad set. A GPO security-filtered to "Everyone + Domain Users" would be flagged as filtered when it shouldn't be. Filed as WI-009.
- `scope_caveats` in topology.py calls `_resolve_som_chain` which returns `None` for a SOM with all-disabled links. This reads as "no caveats" rather than "all GPOs disabled." Edge case, but the wrong answer direction.
- The `_streaming_zip_read` function buffers the full capped content into a `BytesIO` before returning. The size limit IS enforced during decompression, but for a 1.5GB baseline zip that's 1.5GB in memory. Acceptable for the current use case but worth documenting.

**What went well:** The multi-agent dispatch pattern (kimi/glm implement → adversarial-reviewer validates → I fix HIGH findings) caught real issues that single-pass implementation would have missed. Three rounds of review across two waves produced a stronger result than any single agent could have.

**What was awkward:** Two parallel agents (kimi narration + glm security) edited the same working tree simultaneously. The kimi agent reverted pre-existing ingest.py/web/app.py changes it thought were "from an earlier session" — which were actually glm's in-flight changes. Both sets of changes survived by write-ordering luck. The decomposition agent then ran after both, reorganizing the same files. Sequential dispatch with explicit file-ownership boundaries would have been safer.

## Gaps to flag

- **`topology.py:390-410`** — `is_security_filtered` returns `True` for GPOs with empty delegation. Real AD typically inherits a default DACL granting AU Read+Apply. The function should distinguish "no delegation data" from "confirmed filtered."
- **`topology.py:184-200`** — `_resolve_som_chain` returns `None` when all links are disabled. Callers treat `None` as "nothing here" rather than "all disabled," which suppresses caveats that should fire.
- **`cli/_settings.py:108-131`** — The `cmd_settings_at` JSON branch now includes `caveats` (fixed this session), but the text branch's loopback `elif` is dead code when caveats is non-empty (caveats subsumes loopback). Either remove the `elif` or merge the display logic.
- **`detection.py:759`** — `scan_ilt` hard-codes `file="SYSVOL"` in every `IltHit`, discarding the `rel_file` that `_walk_gpp_xml` yields. The doctor finding says "Item-level targeting in SYSVOL" which doesn't tell the user which GPP file.
- **`queries.py:906`** — `stale_gpos` uses `int((now - mod).days / 365)` which ignores leap years. Over short timeframes this truncates correctly, but a 4.7-year-old GPO is reported as "4+ years." Trivially misleading.
- **Parallel-write race risk** — When dispatching multiple agents to the same repo, file-level ownership boundaries are critical. This session had a near-miss where kimi reverted glm's ingest.py changes. Document this as a convention for future sessions.
