## On the work done

**WI-037 (cross-process lock)** was the highest-risk change. The `_FileLock` class combines threading.Lock (fast-path for same-process) with fcntl.flock/msvcrt.locking (cross-process). The adversarial review caught three issues I'd missed: `:memory:` database path needed special handling, `release()` could leak the thread lock if called without successful `acquire()`, and non-OSError exceptions could leak the fd. All three were fixed with an `_acquired` flag and broader exception handling. I'm confident in this — the OS guarantees for flock are well-understood and the test_concurrent_upload_returns_409 test still passes.

**WI-039 (Markdown escaping)** was more nuanced than expected. The initial implementation used `html.escape(quote=False)` everywhere, but the adversarial review identified that values inside backtick code spans need backtick escaping too, and Markdown table cells need pipe escaping. I added `_md_code()` and `_md_table()` helpers. However, the review also identified a display quality issue: `_md_code()` double-escapes HTML entities (since CommonMark doesn't interpret entities inside code spans). This is filed as WI-042 — it's a display issue, not a security issue. The fix would be to NOT apply HTML escaping inside code span contexts, but that requires careful separation of concerns.

**WI-040 (snapshot_diff N+1)** was the most satisfying refactor. The old code ran 8 queries per common GPO; the new code runs 8 × ceil(N/500) queries total. I added chunking (500 per batch) to handle estates with >999 GPOs, which the adversarial review correctly flagged as a SQLite parameter limit risk. The 7 new tests cover all previously-untested batched comparison paths (settings, links, delegation, version skew, no-common-GPOs, settings-appeared-in-one-snapshot).

**WI-038 (ruff expansion)** was the simplest but had the widest blast radius — 15 files auto-fixed. The `B904` ignore was removed after I discovered the codebase already uses `from exc` correctly; the 2 violations were both `raise HTTPException` without `from None` in the web app, which I fixed.

## Gaps to flag

- **`_md_code()` double-escapes HTML entities inside code spans** at `report.py` — values like `<val>` display as `&lt;val&gt;` instead of `<val>` when rendered. Filed as WI-042.
- **Cross-process lock has no multi-process test** at `test_web.py:472` — `test_concurrent_upload_returns_409` only exercises the threading.Lock fast-path, not the file lock. Filed as WI-043.
- **`snapshot_diff` `gpo_link` comparison omits `som_name`** at `snapshot_diff.py:378` — the `gpo_link` table's primary key includes `som_name`, but the batch query selects only `som_path, link_enabled, enforced`. Two links with the same `som_path` but different `som_name` would be deduplicated. Extremely unlikely in practice — `som_name` and `som_path` are derived from the same SOM.
- **`S310` ignore is broad** at `pyproject.toml:54` — `urllib.request.urlopen` is used in `narration.py` (LLM endpoint) and `sinks.py` (Splunk HEC). The URLs come from environment variables, which could be attacker-controlled in a compromised deployment. Documenting the threat model at each call site would be cleaner than a blanket ignore.
- **The `B007` ignore (unused loop variable)** at `pyproject.toml:52` — ruff flags `for i, x in enumerate(items)` when `i` is unused, suggesting `for x in items`. The ignore is a blanket suppression; fixing the individual cases with `_` would be cleaner but was out of scope for this session.
- **Ingest audit log is still written before DB commit** at `app.py:1223-1226` — `_events.append_event` runs before `save_estate` returns. If `save_estate` raises after the audit log, the log claims success. This was noted in the prior session's reflection and remains.
