## On the project

The codebase punches above its weight for a 4-day-old repo: `ruff` + `mypy --strict` clean, 280 tests, a real data model. The thing that impresses me most is the clarity of the architecture — every module has a single job, the CLI is a thin argparse wrapper, and the model/contract (`model.py`, `docs/spec/`) means I can reason about changes without chasing spaghetti. 

The weak spots:
- `queries.py` is ~1900 lines. The Plan 010 roadmap already flags extracting `detection.py`. That's overdue.
- The hand-crafted XML fixture is fine for now, but the adversarial reviewers were right: if `_parse_single_gpo` gains a new field, someone has to remember to update the XML. A `build_fixture.py` generator would amortize that maintenance cost.
- `admx_parser.py` is intentionally lightweight — but it won't handle every ADMX edge case (elements, categories, multi-value definitions). Fine for now, but if Microsoft baseline diff becomes a core feature, it'll need reinforcement.

## On the work done

**What I'm confident in:**
- The synthetic fixture estate. It covers cpassword, MS16-072, version-skew, broken UNC, loopback, block-inheritance, enforced links, precedence conflicts, blocked extensions, disabled-but-populated. The data is internally consistent and the tests are rigorous.
- `snapshot_changelog()` correctly pairs version deltas with per-setting diffs, distinguishing "metadata_only" from "settings_detail" entries.
- `report.py` generates clean, readable Markdown and decent HTML with print-friendly CSS.

**What I'm less confident in:**
- The `settings_at_som` loopback banner is a cosmetic warning. It prints a header but doesn't actually change the folding logic. The charter says "flag, don't simulate" — this is correct — but it means the tool is honest about its limits without solving them. Acceptable, but not satisfying.
- The HTML report has no visual QA. It looks reasonable in a browser, but nobody has printed it to PDF to verify the `@media print` CSS.
- The `admx_gaps` crosswalk relies on `resolve_display_name` matching the exact identity string. In practice, registry key casing and trailing slashes may mismatch. We don't have real ADMX files to calibrate against yet.

**What went well:**
- The agent-review-refine pipeline worked for the fixture. Two reviewers in parallel surfaced real gaps, and the refinement pass addressed them.
- Moving on after each work item landed kept scope tight.

**What was awkward:**
- The coding agent hallucinated completing WI-A.1 (changelog). It reported success, showed fake test output, but wrote nothing to disk. Cost me ~5 minutes of verification + manual implementation. Now I verify every claim with `git status` + `grep`.
- Ruff line-length failures after agent edits slowed me down. The agents didn't internalize the 100-char limit.
